# Docs

This repository follows a "single source of truth" docs layout:

1. **Reference**: normative rules (MUST/MUST NOT/SHOULD). If anything conflicts, Reference wins.
2. **Explanation**: why the rules exist, tradeoffs, and how to think about them.
3. **Process**: lightweight workflow for changing rules and decisions.
4. **ADR**: decision records for architectural choices.
